# Notes — Attention et Transformers

## Introduction aux RNN et leur évolution

Les **RNN (Réseaux de Neurones Récurrents)** sont des modèles qui n'ont plus de raison d'être car ils ont été largement dépassés par les architectures Transformer dans le domaine du **NLP (Natural Language Processing)**.

## Tokenisation et embeddings

### Token
Un **token** = sous-mot ou unité textuelle
Exemple : "write", "written", "writings"
→ "writ" + "e"
→ "writ" + "en"  
→ "writ" + "ings"

### Embedding
Chacun de ces sous-mots a un **embedding** : un vecteur numérique qui représente sa signification dans l'espace vectoriel.

### Padding
**Padding** : ajout de tokens vides pour que toutes les séquences aient la même taille (requis pour le traitement en batch).

## Mécanisme d'attention

### Problématique
Problème que l'attention résout : dans une longue séquence, comment savoir quels mots sont pertinents pour comprendre le mot courant ?

**Exemple** : "Le chat que j'ai vu hier **dort**" → pour comprendre "dort", le mot important est "chat", pas "hier".

### Self-attention (attention sur soi-même)
Pour chaque token, on calcule un score d'attention avec **tous** les autres tokens de la séquence.

#### Trois vecteurs par token :
- **Q** (*Query*) : "qu'est-ce que je cherche ?"
- **K** (*Key*) : "qu'est-ce que j'offre ?"
- **V** (*Value*) : "qu'est-ce que je transmets ?"

#### Formule
Score d'attention = softmax(Q · Kᵀ / √d) × V

Le score est normalisé par √d (dimension) pour éviter que les valeurs soient trop grandes.

### Multi-head attention
On répète le mécanisme d'attention plusieurs fois en parallèle (*heads*), chaque head apprend à regarder des relations différentes dans la séquence.

## Architecture Transformer

### Blocs principaux
Deux blocs principaux :
- **Encoder** : lit la séquence d'entrée et produit une représentation contextuelle
- **Decoder** : génère la séquence de sortie token par token

### Composants de chaque bloc
1. Multi-head attention
2. Feed-forward network (MLP)
3. Layer normalization + résidual connections

### Positional encoding
Les Transformers traitent tous les tokens **en parallèle** → pas d'ordre naturel.
→ On ajoute un vecteur de position à chaque embedding pour injecter l'information d'ordre.

## Modèles basés sur les Transformers

| Modèle | Architecture | Usage |
|--------|-------------|-------|
| BERT | Encoder seul | Classification, NER, extraction |
| GPT | Decoder seul | Génération de texte |
| T5, BART | Encoder-Decoder | Traduction, résumé |
