## Pipeline CI/CD

### Continuous Integration (env dev)
On va lancer les tests automatiquement dès que quelqu'un fait une PR. On ne va pas se baser sur se faire confiance de faire les tests en local. On va le bloquer à l'entrée.
Automated lifecycle.
Continuous integration : automatiser les bonnes pratiques, les checker, et permettre une collaboration cohérente.

What do we need?
- git
- outil d'hébergement de repo
- outil intégration continue : jenkins
- set up un trigger : push, PR, push avec tag ?
- Tests unitaires et tests d'intégration

### Continuous delivery (env preprod)
En plus de la pipeline automatisée on a besoin d'outils de déploiement automatique TOCHECK
Outils : terraform, Tests de qualification

### Continuous deployment (env prod)
Une fois que tout cela est passé - on déploie en prod.
Validation manuelle pour lancer le déploiement.
Outil dépend du service sur lequel on déploie. Ex Kubernetes.

Terraform permet de définir l'infrastructure dans des fichiers YAML. Gestion de l'infrastructure as code. Gestionnaire de version.

## GitHub Actions
Dans ces actions on a des intégrations avec pylint, pytest, terraform, gcp incluses.
Si on clique sur create dans les actions on génère un YAML.
Au préalable dans .github il nous faut un dossier workflows dans lequel on met ces fichiers d'automatisation.
Dans GitHub, on a des actions.
Dans le YAML on a le trigger 'on', niveau de permission, liste d'actions à faire définies dans job.
Installer extension GitHub Actions pour se repérer dans le YAML.
Ensuite on a une liste de steps.
